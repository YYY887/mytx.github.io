XPath 语法详解

一、引言

XPath（XML Path Language）是一种用于在 XML 或 HTML 文档中定位节点的强大语言。在处理 XML 数据或进行网页数据抓取时，XPath 提供了一种简洁而高效的方式来选择文档中的元素和属性。下面将详细介绍 XPath 的语法规则和使用方法。

二、基本概念

（一）节点

在 XML 或 HTML 文档中，元素、属性、文本、注释等都可看作节点。例如，在 HTML 中的 <div> 标签是元素节点，id、class 等是属性节点，标签内的文字属于文本节点。

（二）路径表达式

路径表达式类似于文件系统中的路径，用于在文档中选择节点。它分为绝对路径和相对路径。

三、基本路径表达式

（一）绝对路径和相对路径

  • 绝对路径：从根节点开始的路径，以斜杠 / 开头。例如，/html/body/div 表示从根节点开始，依次选择 html 元素、body 元素，最后选择其中的 div 元素。
  • 相对路径：从当前节点开始的路径，不以斜杠开头。比如，div/p 表示在当前节点下，选择 div 元素中的 p 元素。

（二）节点选择器

  • 节点名称：直接通过节点的名称选择节点。例如，div 会选择文档中所有的 div 元素。
  • 通配符 \*：用于选择所有节点。/* 表示选择根节点下的所有子节点。
  • 属性选择器：根据节点的属性来选择节点。例如，//div[@id='myDiv'] 会选择文档中所有 id 属性值为 myDiv 的 div 元素。

（三）轴（Axes）

轴用于定义相对于当前节点的节点集合，以下是一些常用的轴：

  • child：选择当前节点的子节点，这是默认轴，通常可以省略。如 div/child::p 与 div/p 等价。
  • parent：选择当前节点的父节点。例如，p/parent::div 表示选择 p 元素的父 div 元素。
  • descendant：选择当前节点的所有后代节点（包括子节点、孙节点等）。div/descendant::p 表示选择 div 元素下的所有 p 元素。
  • ancestor：选择当前节点的所有祖先节点（如父节点、祖父节点等）。p/ancestor::div 表示选择 p 元素的所有祖先 div 元素。
  • following - sibling：选择当前节点之后的所有兄弟节点。p/following - sibling::div 表示选择 p 元素之后的所有 div 兄弟节点。
  • preceding - sibling：选择当前节点之前的所有兄弟节点。p/preceding - sibling::div 表示选择 p 元素之前的所有 div 兄弟节点。

四、谓语（Predicates）

谓语用于过滤节点集合，它是方括号 [] 中的表达式。以下是一些常见的使用示例：

  • //div[1]：表示选择所有 div 元素中的第一个。
  • //div[@class='myClass']：表示选择所有 class 属性值为 myClass 的 div 元素。
  • //div[text()='Hello']：表示选择所有文本内容为 Hello 的 div 元素。

五、函数

XPath 提供了一些内置函数，用于处理节点和字符串。以下是一些常用的函数：

  • text()：返回节点的文本内容。例如，//div/text() 表示选择所有 div 元素的文本内容。
  • contains()：检查字符串是否包含指定的子字符串。例如，//div[contains(@class, 'myClass')] 表示选择所有 class 属性值包含 myClass 的 div 元素。
  • starts - with()：检查字符串是否以指定的子字符串开头。例如，//div[starts - with(@id, 'prefix')] 表示选择所有 id 属性值以 prefix 开头的 div 元素。

六、示例

以下是一个简单的 HTML 文档和一些 XPath 表达式的示例：

html

  • //div：选择所有的 div 元素。
  • //div/p：选择所有 div 元素下的 p 元素。
  • //div[@id='div1']：选择 id 属性值为 div1 的 div 元素。
  • //div[@id='div1']/p：选择 id 属性值为 div1 的 div 元素下的 p 元素。
  • //p[text()='Paragraph 1']：选择文本内容为 Paragraph 1 的 p 元素。
